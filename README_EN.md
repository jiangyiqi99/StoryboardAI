# Storyboard AI Editor

[中文](README.md) | [English](README_EN.md)

An AI-native local non-linear video editor scaffold built with Electron, React, TypeScript, Vite, and an FFmpeg/libav sidecar. The current project establishes the application structure, core domain types, IPC flow, and packaging pipeline; it is not yet a complete production video editor.

## Goals

- Desktop-first, cross-platform Electron application.
- Local projects stored in `.aivproj/` directories with `project.json` as the source of truth.
- Timeline-first editing model: tracks, clips, assets, selections, and edit history are core entities.
- Separate preview, frame-extraction, and rendering responsibilities.
- AI workflows for storyboard-to-video generation and replacement of a selected timeline range.
- Non-destructive editing: clips reference original media instead of overwriting it.

## Packaging

Each distribution command builds the Go/libav sidecar, stages its FFmpeg runtime libraries, builds the Electron app, and writes packages to `release/`.

```bash
npm run dist:macarm64
npm run dist:macx64
npm run dist:win
npm run dist:linux
```

The sidecar must be built on the target operating system and CPU architecture. GitHub Actions builds Windows x64, macOS arm64, macOS x64, and Linux x64. Every push creates build artifacts; pushing a `v*` tag also creates a GitHub Release.

Release builds embed the sidecar and its resolved FFmpeg runtime dependencies, so end users do not need a separate FFmpeg installation. macOS notarization and Windows code signing are not configured, so generated packages are unsigned.

Because the bundled FFmpeg build is GPL, a binary release must also make available the exact corresponding FFmpeg source, build configuration, and the complete corresponding source for this project.

## Architecture overview

```text
Renderer UI
  -> preload API
  -> Electron IPC
  -> Main-process handlers
  -> Project System / Media Engine / AI Orchestrator
  -> local project files and media outputs
```

- `src/renderer`: editor UI and user interaction.
- `src/main`: Electron main process, IPC handlers, local resource access, and native-runtime management.
- `src/project-system`: `.aivproj` paths, schema, assets, and project-file persistence.
- `src/media-engine`: preview, frame, and render-engine contracts.
- `src/ai-orchestrator`: provider-neutral AI video workflows.
- `src/editing`: timeline editing commands.
- `native/libav`: Go/libav sidecar and its JSON-RPC protocol.
